# Настройка Frontend

## Проблемы с импортами в IDE

Если вы видите ошибки типа "Cannot find module 'react'" в IDE, это нормально - зависимости еще не установлены.

## Решение

### Вариант 1: Установить зависимости локально (для разработки)
```bash
cd frontend
npm install
```

После этого ошибки в IDE должны исчезнуть.

### Вариант 2: Использовать Docker (рекомендуется)
Зависимости автоматически установятся при сборке Docker образа:
```bash
docker-compose up -d frontend
```

## Проверка

После установки зависимостей все импорты должны работать корректно:
- ✅ `react` и `react-dom`
- ✅ `react-router-dom`
- ✅ `@mui/material`
- ✅ `axios`

## Структура импортов

Все импорты используют относительные пути:
- `./context/AuthContext` - из src/
- `../api/auth` - из подпапок
- `@mui/material` - из node_modules (после npm install)

## Примечание

Ошибки TypeScript в IDE не влияют на работу приложения в Docker, так как зависимости устанавливаются при сборке образа.
